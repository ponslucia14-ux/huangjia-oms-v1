# 凰家项目工作状态协议

## 《凰家开工》与《凰家收工》正式定义 V1.0

- 版本：V1.0
- 适用项目：凰家母婴空间数字化建设、凰家 OMS、凰家大脑
- 适用人员：石磊、张照南及所有凰家项目参与人员
- 生效日期：2026-07-12

## 1. 目的

“凰家开工”和“凰家收工”是项目生命周期管理指令。目标是在不同电脑、环境、聊天窗口和时间阶段之间准确恢复项目上下文，不丢失成果、不重复劳动、不产生版本混乱。

最高原则：真实、连续、可追溯、可恢复；不虚假汇报、不丢失历史、不重复建设。

## 2. 凰家开工

“凰家开工”表示开启新的工作周期，执行“恢复、检查、继续”，不得在不了解当前版本时直接开发。

必须确认：

1. 当前项目、代码版本和文档版本。
2. 最近一次工作记录和收工状态。
3. 已完成、待完成、老板已确认及禁止修改事项。
4. 当前电脑环境、代码目录、Git分支、最近提交和GitHub同步状态。

输出必须包含日期、项目、版本、最近收工状态、已完成事项、当前任务、Git和GitHub状态。

## 3. 凰家收工

“凰家收工”表示停止扩展，保存成果，冻结当前状态，为下一次开工提供准确入口。

必须停止新需求扩展、架构调整和非必要优化，并记录：

1. 今日完成模块、修改文件、功能、测试结果和重要决定。
2. 今日未完成事项、等待确认事项、技术风险和下一步计划。
3. 当前项目阶段、代码状态和文件状态。

## 4. GitHub同步规则

代码或文件发生变化时必须：

1. 查看 `git status` 并确认修改范围。
2. 提交真实成果。
3. 推送至GitHub。
4. 返回仓库、分支、提交编号、推送结果和最终工作区状态。

没有文件变化时必须明确说明“今日无代码变更，无commit”，禁止制造无意义提交。

## 5. 跨电脑规则

- GitHub是正式代码来源，项目文档是正式知识来源，聊天记录不是最终版本。
- 任何电脑开工前必须先同步最新状态。
- 禁止两个电脑产生不同版本后强行覆盖或合并。
- 遇到远端领先和本地未提交并存时，必须先保护本地成果，再同步、检查冲突和验证。

## 6. 真实性与追溯

- 没有测试结果，不说完成。
- 没有commit，不说同步。
- 没有push，不说上传。
- 所有重要修改和决定必须保留原因、确认人、时间及历史记录。

## 8. 最终解释

- 凰家开工 = 恢复项目。
- 凰家收工 = 保存项目。

本协议自2026-07-12起作为凰家项目长期工作状态切换标准执行。
