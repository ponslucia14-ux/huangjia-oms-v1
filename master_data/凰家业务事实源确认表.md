## 二、确认规则

1. 四份原始业务 Excel 的文件来源默认可信，不重复确认文件来源。
2. 必须扫描全部 Sheet，并基于字段、时间、记录状态和业务用途生成候选。
3. 禁止仅按文件名、Sheet 名称或顺序选择 Current。
4. 当且仅当一个文件存在唯一 Current 候选，且质量检查为 PASS 时，自动进入 Current。
5. 多个 Current 候选、用途冲突、字段冲突或数据异常才进入人工确认。
6. 人工确认必须记录文件、Sheet、负责人、确认时间、用途和是否进入 Current。
7. 禁止为了上线强行准入。
8. 负责人必须使用 EMP 真实身份签认；昵称、岗位代称和虚拟用户无效。

## 九、Sheet 语义记忆机制

首次业务确认通过后，OMS 保存：

- `source_file_pattern`
- `source_sheet`
- `domain`
- `fact_type`
- `owner`
- `quality_result`
- Sheet 集合
- 字段指纹
- 结构指纹
- 确认时间

后续同结构文件优先使用历史确认规则：

```text
文件模式匹配
+ Sheet一致
+ Domain一致
+ Sheet集合一致
+ 字段指纹一致
+ 结构指纹一致
+ 数据质量未下降
→ AUTO_RECOGNIZED
```

以下任一变化强制重新进入 REVIEW：

- `sheet_added`
- `sheet_removed`
- `fields_changed`
- `structure_changed`
- `data_quality_declined`
- 同一文件模式出现多个冲突记忆

语义记忆只复用业务用途，不跳过 Data Quality 检查，也不直接写 Truth Source。

当前状态：首次四域确认尚未完成，`SheetSemanticMemory` 中不得写入任何生产确认记录。

## 十、P0.17.3 语义记忆生命周期

语义记忆采用追加版本制：

- `memory_version`：规则每次确认或修订生成新版本，禁止覆盖旧版本。
- `CONFIRMED`：业务负责人已确认，可在结构和质量均稳定时自动复用。
- `TEMPORARY`：临时候选，后续导入仍必须进入REVIEW。
- `DEPRECATED`：失效历史规则，仅供追溯，不再参与自动识别。

状态路径：`TEMPORARY -> CONFIRMED -> DEPRECATED`，或`CONFIRMED -> DEPRECATED`。旧版本及状态变更必须保留Audit。

Sales Current、Finance Current、Room Current、Actual Stay当前仍未完成首次Sheet业务签认，因此四条正式记忆均为`NOT CREATED`，不得以临时规则替代确认。
