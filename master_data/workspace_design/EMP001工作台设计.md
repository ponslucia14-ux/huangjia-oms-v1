# EMP001 石磊工作台设计

## 1. 角色定位

主理人，承担全局经营判断、跨域例外裁决和重大事项批准；不作为销售、财务、房态的日常录入人。

## 2. 工作台树形结构

```text
首页
├─ 今日决策
├─ 经营状态
└─ 经营风险
销售经营
├─ 签约结果
├─ 合同与收款状态
└─ 例外审批
资金经营
├─ 日结状态
├─ 待收待付
└─ 财务异常
运营经营
├─ 当前入住
├─ 当前房态
└─ 服务异常
组织执行
├─ 今日责任人
└─ 未闭环事项
审计追溯
```

## 3. 首页 Dashboard

- Current 未初始化时只显示初始化进度，不显示历史数字。
- 初始化后显示今日新增事实、已确认资金状态、实际入住与房态、待决策风险。
- 每个数字必须显示责任人、更新时间和 Current 版本。

## 4. Action 待办

- 金额、退款、房间冲突等重大例外审批。
- 跨部门责任冲突裁决。
- 超时未闭环事项和高风险异常确认。
- 数据初始化或新 Snapshot 激活批准。

## 5. Input 录入项

- 审批决定、驳回原因、例外授权范围和期限。
- 经营目标与当天优先级。
- 不录入日常流水、客户合同、房态和入住明细。

## 6. Output 产生事实

- Approval Decision、Owner Decision、Risk Acknowledgement。
- Snapshot Activation Decision。
- 所有输出带 EMP001、时间、原因、Audit、Event。

## 7. 权限范围

全域只读；重大例外审批；跨域裁决。查看不产生写入，批准不等于自动执行。

## 8. 不应该看到的内容

- 旧 Excel Current、技术模块、Engine、schema、runtime。
- 普通员工录入按钮和无责任归属的审批。
- CRM 线索、签约前跟进、照护师登录管理。

## 每日使用答案

打开后能直接知道：今天要裁决什么、经营事实是否已确认、哪里有风险、由谁处理。
