# EMP009 尚丽娜工作台设计

## 1. 角色定位

管家，负责入住资料准备、到馆事实采集、在住服务跟进和运营异常上报；不负责最终发布 Room Current。

## 2. 工作台树形结构

```text
首页
├─ 今日接待
├─ 在住服务待办
└─ 异常上报
入住准备
├─ 已确认入住计划
├─ 资料检查
└─ 到馆登记
在住服务
├─ 我的在住客户
├─ 服务事项
└─ 需求/异常
出馆协同
├─ 出馆准备
└─ 资料交接
我的记录
```

## 3. 首页 Dashboard

显示本人负责的今日接待、资料缺口、在住服务任务和待店总确认事项，不显示全局财务和其他员工客户隐私。

## 4. Action 待办

- 核对客户到馆、身份资料、套餐和服务需求。
- 提交入住确认候选、出馆候选和房间异常。
- 完成本人服务事项并记录结果。

## 5. Input 录入项

实际到馆时间、客户资料、建议房号、服务需求、异常、出馆准备结果；关键入住事实提交给 EMP008 确认。

## 6. Output 产生事实

Butler Intake Record、Stay Confirmation Request、Service Record、Exception Report、Checkout Preparation。

## 7. 权限范围

仅查看和维护本人责任客户；可提交但不可自行批准实际入住、调房或出馆；不能确认资金。

## 8. 不应该看到的内容

全公司财务、合同金额修改、房态最终发布、其他员工无关客户、旧计划冒充实际入住。

## 每日使用答案

打开后知道今天接谁、服务谁、缺什么资料、需要向店总确认什么。
