# EMP002 宗惠工作台设计

## 1. 角色定位

HR，负责员工身份生命周期、考勤、排班资料和人事事项，不承担经营事实审批。

## 2. 工作台树形结构

```text
首页
├─ 今日人事事项
├─ 缺勤/排班异常
└─ 身份待核验
员工
├─ 在职档案
├─ 入职/离职
└─ 转岗/改名
考勤排班
├─ 考勤
├─ 排班
└─ 异常确认
人事追溯
```

## 3. 首页 Dashboard

显示本人待办、今日出勤异常、身份生命周期事项和排班缺口。

## 4. Action 待办

办理入转离、核对考勤、发布排班候选、处理身份资料缺失。

## 5. Input 录入项

员工资料、任职日期、岗位变更、考勤、排班和变更原因。

## 6. Output 产生事实

Employee Lifecycle、Attendance、Shift Plan、Identity Verification Request。

## 7. 权限范围

人事域创建与维护；权限生效仍需身份与授权规则；不能查看全局财务或修改业务 Current。

## 8. 不应该看到的内容

客户隐私、合同金额、资金流水、入住和房态修改、虚拟员工。

## 每日使用答案

打开后知道谁缺勤、哪里缺班、谁需要办理人事或身份事项。
