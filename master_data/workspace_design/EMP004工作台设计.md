# EMP004 刘晶工作台设计

## 1. 角色定位

出纳，负责真实收付款、待收待付和每日资金日结的录入；录入不等于最终复核。

## 2. 工作台树形结构

```text
首页
├─ 今日资金动作
├─ 待复核
└─ 到期提醒
收款
├─ 新增收款
├─ 待收
└─ 收款凭证
付款
├─ 新增支出
├─ 待付
└─ 付款凭证
日结
├─ 资金余额
├─ 提交复核
└─ 差异处理
财务追溯
```

## 3. 首页 Dashboard

显示今日已录入收支、待收待付、未提交日结和被驳回记录；未复核金额明确标记“待确认”。

## 4. Action 待办

录入真实到账/付款、补充凭证、处理到期项、提交日结、修正被驳回候选。

## 5. Input 录入项

日期、类型、金额、对方、关联客户/合同、用途、支付方式、凭证、到期日、原因。

## 6. Output 产生事实

Payment Candidate、Expense Candidate、Receivable/Payable、Daily Cash Closing；经 EMP003 复核后才成为 Finance Current。

## 7. 权限范围

可创建和修正本人未发布记录；不能复核自己、修改合同金额、覆盖已确认流水或物理删除。

## 8. 不应该看到的内容

销售客户跟进、房态编辑、未授权工资明细、把历史汇总当今日资金。

## 每日使用答案

打开后知道今天收了什么、付了什么、还差哪些凭证和哪些记录等待复核。
