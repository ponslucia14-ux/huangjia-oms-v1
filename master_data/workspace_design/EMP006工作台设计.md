# EMP006 杨欢欢工作台设计

## 1. 角色定位

销售顾问及销售业务主责，负责签约后客户、合同和入住计划的业务录入；凰家不采用线索与签约前跟进流程。

## 2. 工作台树形结构

```text
首页
├─ 今日签约结果
├─ 合同待补全
└─ 待财务确认
签约客户
├─ 新建签约客户
├─ 我的签约客户
└─ 客户状态
合同
├─ 新建合同
├─ 金额/套餐
└─ 变更申请
收款协同
├─ 收款主张
└─ 财务确认结果
入住计划
└─ 交接店总/管家
```

## 3. 首页 Dashboard

显示本人今日签约、合同缺失项、待财务复核金额和待交接入住计划，不显示线索漏斗。

## 4. Action 待办

完成签约资料、提交合同金额复核、提交收款主张、创建入住计划、处理驳回和交接。

## 5. Input 录入项

客户正式资料、合同号、签约日期、套餐、合同金额、销售人员、预计入住信息、收款主张和凭证引用。

## 6. Output 产生事实

Customer、Contract Candidate、Contract Stay Plan、Sales Result、Payment Claim；到账事实只能由财务确认。

## 7. 权限范围

维护本人及获授权销售记录；合同金额需 EMP003 复核；特殊优惠/退款需审批；不能确认实际到账或实际入住。

## 8. 不应该看到的内容

线索、客户跟进、转化漏斗、全局财务流水、房态修改、计划入住冒充 Actual Stay。

## 每日使用答案

打开后知道今天签了什么、资料是否完整、钱是否被财务确认、哪些计划需要交接运营。
