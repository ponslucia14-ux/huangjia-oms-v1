# EMP010 陈晶辉工作台设计

## 1. 角色定位

产护总监，负责产护服务计划、任务分配、质量检查和异常闭环，不负责入住与房态最终确认。

## 2. 工作台树形结构

```text
首页
├─ 今日产护任务
├─ 服务风险
└─ 待确认异常
服务计划
├─ 在住客户需求
├─ 产护计划
└─ 人员安排
执行质量
├─ 任务完成
├─ 交接
└─ 异常处理
产护追溯
```

## 3. 首页 Dashboard

显示今日服务对象、未分配任务、逾期任务、风险和交接事项。

## 4. Action 待办

制定计划、分配执行、确认完成、处理护理异常和跨班交接。

## 5. Input 录入项

服务需求、计划、执行人、时间、完成结果、异常等级和处理记录。

## 6. Output 产生事实

Care Plan、Care Task、Service Completion、Care Exception、Quality Review。

## 7. 权限范围

产护域及必要的在住客户最小信息；不能修改入住、房号、合同或资金事实。

## 8. 不应该看到的内容

客户无关财务、销售编辑、房态发布、照护师登录管理模块。

## 每日使用答案

打开后知道今天服务谁、谁执行、哪里有风险、哪些服务尚未闭环。
