# EMP003 张敬东工作台设计

## 1. 角色定位

会计，负责合同金额、到账、支出、日结与结算复核，保证 Finance Current 的确认责任与录入责任分离。

## 2. 工作台树形结构

```text
首页
├─ 待复核资金
├─ 差异与异常
└─ 到期事项
复核中心
├─ 收款复核
├─ 支出复核
├─ 合同金额复核
└─ 日结复核
对账结算
├─ 待收待付
├─ 对账
└─ 结算归档
财务追溯
```

## 3. 首页 Dashboard

显示待复核数量、金额差异、凭证缺失、逾期未收未付和当日关账状态。

## 4. Action 待办

批准/驳回出纳提交，核对合同与到账，处理差异，确认日结和结算。

## 5. Input 录入项

复核结论、差异说明、会计分类、结算期间和驳回原因；不得代替出纳篡改原记录。

## 6. Output 产生事实

Finance Review、Payment Confirmation、Expense Confirmation、Reconciliation Result、Finance Current。

## 7. 权限范围

全财务域查看与复核；不能审核自己创建的重大记录；不能修改销售和运营事实。

## 8. 不应该看到的内容

房态操作、入住发布、销售跟进、无凭证金额直接进入 Current。

## 每日使用答案

打开后知道哪些钱需要确认、哪里对不上、当天是否可以关账。
