# OMS实际入口验收规则

版本：V1.0  
生效日期：2026-07-13  
确认人：石磊  
适用范围：OMS 页面、功能、样式、权限、菜单、交互及前端发布

## 一、唯一最终验收入口

最终验收入口只能是：

1. 飞书电脑端真实入口。
2. 飞书手机端真实入口。

飞书手机端实际效果为强制验收项。

以下内容只能作为开发验证，不能作为最终验收：

- 本地浏览器。
- 开发环境页面。
- 模拟器。
- 浏览器开发者工具。
- 自动截图。
- 代码测试结果。

## 二、每次改动的强制流程

1. 完成代码修改。
2. 完成自动测试。
3. 部署到飞书实际可访问的验收环境。
4. 使用对应员工的真实飞书身份进入。
5. 在飞书电脑端核对菜单、页面、权限和交互。
6. 在飞书手机端核对菜单、页面、权限和交互。
7. 保存飞书电脑端实际截图。
8. 保存飞书手机端实际截图或录像。
9. 记录实际入口地址或进入路径及部署版本。
10. 等待石磊本人在飞书手机端确认。

## 三、判定为未完成的情况

出现以下任一情况，一律视为未完成：

- 本地页面已修改，但飞书入口没有变化。
- 飞书电脑端正常，但飞书手机端异常。
- 测试通过，但真实飞书无法使用。
- 自动截图正常，但石磊实际进入看到的不是同一版本。
- 使用管理员身份代替对应员工真实身份验收。
- 只验证页面打开，没有验证菜单和操作。
- 改动没有部署到飞书实际入口。
- 没有石磊飞书手机端确认记录。

## 四、每次提交必须附带

- 本次修改内容。
- 飞书电脑端实际截图。
- 飞书手机端实际截图或录像。
- 使用的真实员工身份。
- 实际进入路径。
- 当前部署版本。
- 石磊是否已实机确认。

## 五、统一验收状态

| 状态 | 含义 |
|---|---|
| 开发中 | 尚未完成代码或部署 |
| 代码已完成 | 代码和自动测试完成，但不代表验收完成 |
| 已部署待验收 | 已进入飞书可访问环境，等待真实入口核对 |
| 飞书电脑端已验收 | 电脑端真实入口通过，手机端尚未确认 |
| 飞书手机端待石磊确认 | 手机端已取得真实入口证据，等待石磊本人确认 |
| 完成 | 石磊本人在飞书手机端看到并确认正确效果 |

未经飞书手机端真实验收，不得标记通过，不得进入正式版本。
